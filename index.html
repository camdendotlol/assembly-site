Hello from Assembly

This website is being served by an Assembly web server called rwasa.

I think that's pretty cool.

On the backend, it's lightning fast. But since you also have to wait for this page to transfer over the internet, there are diminishing returns on server speed for simple sites like this.

While you might not notice the exceptional speed from your end, the rwasa server program launches instantly. According to the top utility, rwasa uses just a few kilobytes of memory while running. And it doesn't require any extra libraries. Compare that to the typical server frameworks in use today, which require all kinds of scripting languages, packages, and other nonsense just to run.

rwasa even supports SSL, as you can see by the beautiful green lock icon in your URL bar.

Of course, there is the issue that my lack of Assembly knowledge makes this server a black box for me, so I can't easily dig into it if I need to make changes or achieve an understanding of its internals.

If I were to make a performance-critical backend for a serious project, I would probably go with a Rust framework like Rocket or Actix. Most of the time, however, a boring old JavaScript or Python server is just fine.

In theory, however, there's no reason you couldn't serve a full-scale production application with this and see some nice serverside performance improvements.
